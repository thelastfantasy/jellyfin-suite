# Feature Specification: Jellyfin 最近播放视图插件

**Feature Branch**: `001-recents-view-plugin`
**Created**: 2026-05-09
**Status**: Draft
**Input**: User description: "Jellyfin 最近播放视图插件 - 在 Jellyfin Web 中创建一个类似"文件夹"视图的自定义插件页面，展示当前用户的最近播放记录，支持多维度分组（天/周/月/季度/年）、多种排序方式、重复条目开关，以及分页浏览历史记录。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 浏览最近播放记录（默认视图）(Priority: P1)

用户打开 Jellyfin Web，从侧边菜单或插件入口进入"最近播放"页面，看到当前用户最近播放的视频内容，以默认的"按周"分组方式呈现，每页展示最近 13 周的播放记录。每个条目显示缩略图、标题和最后播放时间。

**Why this priority**: 这是插件的核心功能。没有这个，插件毫无价值。所有其他功能都依赖于此基础显示能力。

**Independent Test**: 可通过进入插件页面、确认条目按周分组显示、且仅包含当前登录用户的播放记录来独立测试。

**Acceptance Scenarios**:

1. **Given** 用户已登录 Jellyfin Web 且有播放历史，**When** 用户打开最近播放插件页面，**Then** 看到按周分组的视频播放记录，每组显示该周的周标签，条目显示缩略图、标题、最后播放时间
2. **Given** 用户处于默认分组（按周），**When** 页面加载完成，**Then** 仅显示最近 13 周的数据，页面底部有翻页控件可查看更早的记录
3. **Given** 用户没有任何播放历史，**When** 打开插件页面，**Then** 显示空状态提示，如"暂无播放记录"

---

### User Story 2 - 切换分组方式 (Priority: P2)

用户可以在"按天"、"按周"、"按月"、"按季度"、"按年"之间切换分组方式，页面内容随之重新排列，分页范围也随之调整。季度分组使用中文季节名称（冬/春/夏/秋）。

**Why this priority**: 自定义分组是本插件区别于 Jellyfin 内置功能的核心差异点，优先级仅次于基础显示。

**Independent Test**: 通过依次选择每种分组方式，确认条目按对应时间维度正确分组、组标签格式正确、每页显示条目数量符合对应分页规则来测试。

**Acceptance Scenarios**:

1. **Given** 用户在插件页面，**When** 选择"按天"分组，**Then** 记录按播放日期（天）分组，每组标签显示具体日期（如"2026年5月9日"），每页显示最近 30 天
2. **Given** 用户选择"按季度"分组，**When** 页面更新，**Then** 组标签显示季节名称和年份（如"2026年春季 (4-6月)"），每页显示最近 2 个季度
3. **Given** 用户选择"按月"分组，**When** 翻到下一页，**Then** 显示再前 6 个月的数据，并可继续翻页

| 分组方式 | 每页范围    | 组标签示例                  |
|---------|-----------|--------------------------|
| 按天    | 最近 30 天  | 2026年5月9日               |
| 按周    | 最近 13 周  | 2026年第19周 (5月4日-5月10日) |
| 按月    | 最近 6 个月 | 2026年5月                 |
| 按季度  | 最近 2 季度 | 2026年春季 (4-6月)          |
| 按年    | 最近 1 年   | 2026年                   |

---

### User Story 3 - 切换排序方式 (Priority: P2)

用户可以在同一分组内按不同维度排序条目：标题（字母序）、最后播放时间、收藏时间、发行时间、添加到媒体库时间。

**Why this priority**: 排序与分组共同决定用户如何高效浏览记录，是主要功能的重要组成部分。

**Independent Test**: 通过选择每种排序方式，确认同一分组内的条目按对应字段重新排列来测试。

**Acceptance Scenarios**:

1. **Given** 用户在"按周"分组视图，**When** 选择"按标题"排序，**Then** 每个周组内的条目按标题字母序（A→Z）排列
2. **Given** 用户选择"按发行时间"排序，**When** 页面更新，**Then** 每组内条目按媒体 PremiereDate（精确到日）降序排列；无 PremiereDate 的条目排末尾
3. **Given** 用户选择"收藏时间"排序，**When** 页面更新，**Then** 有收藏记录的条目按收藏时间降序排在前面，从未收藏（FavoritedAt 为 null）的条目排在后面；同为未收藏的条目按播放时间降序作为次级排序

---

### User Story 4 - 重复条目显示开关 (Priority: P3)

用户可以开启/关闭"显示重复条目"开关。开启时，同一媒体在同一分组周期内的每次播放记录都单独显示（带各自的播放时间）；关闭时，同一媒体在同一分组周期内只显示最后一次播放记录。

**Why this priority**: 这是一个细化功能，增强灵活性，但不影响核心体验。

**Independent Test**: 准备一个同一视频被多次播放的账号，分别在开关开启/关闭时验证该视频在同一分组内出现的次数。

**Acceptance Scenarios**:

1. **Given** 同一视频在本周被播放了 3 次（昨晚、今天早上、今天下午），**When** 开启"显示重复条目"，**Then** 该视频在本周组内显示 3 条记录，每条显示各自的播放时间
2. **Given** 同样场景，**When** 关闭"显示重复条目"，**Then** 该视频在本周组内只显示最后一次（今天下午）的播放记录
3. **Given** 用户切换此开关，**When** 开关状态改变，**Then** 页面立即刷新，无需手动重载

---

### User Story 5 - 切换视图模式（列表/缩略图/海报）(Priority: P2)

用户可以在工具栏切换三种视图模式：
- **缩略图**（默认）：16:9 横向图片 + 标题 + 播放时间，网格排列
- **海报**：2:3 竖向海报图片 + 标题 + 播放时间，网格排列（适合有海报封面的内容）
- **列表**：紧凑行列表，显示小缩略图 + 标题 + 媒体类型 + 播放时间，每行一条记录

**Why this priority**: 不同用户对内容展示方式有不同偏好；不同媒体类型（动漫海报 vs 视频截帧）在不同视图下效果差异明显。

**Independent Test**: 依次切换三种模式，确认卡片布局、图片比例和列表样式正确切换；视图设置在页面刷新后保持。

**Acceptance Scenarios**:

1. **Given** 用户在缩略图模式，**When** 切换至海报模式，**Then** 卡片变为 2:3 纵向比例，图片使用 Primary 封面图
2. **Given** 用户切换至列表模式，**When** 页面更新，**Then** 每条记录独占一行，包含小缩略图、标题、媒体类型标签和播放时间
3. **Given** 用户切换视图模式，**When** 刷新页面，**Then** 视图模式设置已持久化，恢复上次选择

---

### User Story 6 - 通过第三方仓库安装与更新插件 (Priority: P2)

用户在 Jellyfin 控制台的插件目录中，添加本插件的第三方仓库地址后，可以像安装官方插件一样安装、查看版本信息并接收后续更新。

**Why this priority**: 插件如果只能手动复制 DLL 安装，对普通用户门槛极高。标准仓库分发机制是插件可用性的重要组成部分。

**Independent Test**: 在全新 Jellyfin 实例中，仅通过添加仓库 URL 和点击安装，验证插件能完整安装并正常运行。

**Acceptance Scenarios**:

1. **Given** 用户在 Jellyfin 控制台 → 插件 → 仓库，**When** 添加本插件的 manifest 仓库 URL，**Then** 插件目录中显示"Jellyfin Recents"，包含版本号、描述和安装按钮
2. **Given** 插件已安装，**When** 开发者发布新版本，**Then** Jellyfin 控制台的插件更新提示中出现新版本，用户可一键更新
3. **Given** 新版本与当前 Jellyfin 服务器版本不兼容，**When** 用户查看插件目录，**Then** 不兼容版本不显示为可安装选项

---

### User Story 6 - 视频/音频内容过滤 (Priority: P3)

用户可以在"仅视频"、"仅音频"或"全部"之间切换显示内容类型，默认只显示视频。

**Why this priority**: 按用户需求为次要功能，默认行为已满足主要使用场景。

**Independent Test**: 账号有视频和音频播放记录，切换过滤选项后验证列表中只出现对应类型的条目。

**Acceptance Scenarios**:

1. **Given** 用户打开插件页面（默认状态），**When** 页面加载，**Then** 仅显示视频类型（电影、剧集）的播放记录，音频记录不显示
2. **Given** 用户切换到"仅音频"，**When** 页面刷新，**Then** 仅显示音乐/有声书等音频条目
3. **Given** 用户切换到"全部"，**When** 页面刷新，**Then** 视频和音频条目都显示，但在视觉上有区分标识

---

### Edge Cases

- 用户播放记录极多（数千条）时，翻页逻辑是否能正确按时间窗口分页，不出现条目重复或遗漏？
- 分组跨年时（如"按周"分组，某周跨越 12月31日和1月1日），该周归属于哪一年的哪一页？（假设：按该周多数天数所在年份计算，或按该周起始日所在年份计算）
- 用户在某分组周期内没有播放记录时，该组是否显示？（假设：跳过空组，不显示空组标题）
- "收藏时间"排序时，未收藏（FavoritedAt 为 null）的条目排在末尾，同为已收藏的按 FavoritedAt 降序；同为未收藏的按播放时间降序作为次级排序
- 网络中断导致 API 请求失败时的错误提示与重试机制
- 插件 manifest 中 targetAbi 版本范围如何界定？（假设：以 Jellyfin 10.8.0 为最低兼容版本）

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统必须在 Jellyfin Web 界面中提供可访问的"最近播放"插件入口（如侧边栏菜单项）
- **FR-002**: 系统必须仅展示当前登录用户的播放历史记录，不得显示其他用户数据
- **FR-003**: 系统必须支持五种分组方式：按天、按周（默认）、按月、按季度、按年，分组基准为媒体的播放时间
- **FR-004**: 季度分组必须使用中文季节名称：1-3月为冬季，4-6月为春季，7-9月为夏季，10-12月为秋季
- **FR-005**: 系统必须按以下规则分页展示历史记录：天→30天/页，周→13周/页，月→6个月/页，季度→2季度/页，年→1年/页
- **FR-006**: 用户必须能通过翻页控件查看超出当前页范围的更早历史记录
- **FR-007**: 系统必须支持五种排序方式（作用于分组内部排序）：标题、最后播放时间、收藏时间（最近收藏排前，未收藏排末尾）、发行时间（使用 `PremiereDate` 精确到日，无值时排末尾）、添加到媒体库时间
- **FR-016**: 系统必须支持三种视图模式：缩略图（16:9 横向图片网格，默认）、海报（2:3 竖向海报网格）、列表（紧凑行列表）；可通过工具栏切换并持久化至 localStorage
- **FR-008**: 系统必须提供"显示重复条目"开关；开启时显示同一媒体在同一分组内的每次播放记录；关闭时每个分组内同一媒体只显示最后一次记录
- **FR-009**: 每条播放记录必须显示：媒体缩略图、媒体标题、播放时间（独占一行）
- **FR-010**: 系统必须默认只显示视频类型内容（电影、剧集），并支持切换至仅音频或全部类型
- **FR-011**: 切换分组方式、排序方式、内容类型过滤、重复条目开关时，页面必须自动刷新显示结果
- **FR-012**: 当无播放记录时，页面必须显示友好的空状态提示
- **FR-013**: 用户的分组方式、排序方式、内容过滤、重复条目开关状态必须在同一会话内保持，页面刷新后恢复上次设置
- **FR-014**: 插件必须通过标准 Jellyfin 第三方插件仓库机制发布，用户只需在 Jellyfin 控制台添加仓库 URL 即可安装
- **FR-015**: 插件发布新版本时，已安装用户的 Jellyfin 控制台必须能通过内置更新机制检测并提示更新

### Key Entities

- **PlayRecord（播放记录）**: 代表用户播放某一媒体的一次行为，核心属性：媒体ID、媒体标题、缩略图、播放时间（PlayedAt）、媒体类型（视频/音频）、发行时间（`releaseDate: Date | null`，来自 `PremiereDate`，精确到日）、添加时间（`addedDate: Date | null`）、收藏时间（`favoritedAt: Date | null`）
- **PlayHistoryRecord（持久化播放事件）**: 存储于 SQLite 数据库，记录每次播放开始事件：UserId、ItemId、PlayedAt、MediaType
- **FavoriteRecord（持久化收藏事件）**: 存储于 SQLite 数据库，记录用户收藏某媒体的最新时间：UserId、ItemId、FavoritedAt（null 表示已取消收藏）
- **GroupedPage（分组分页）**: 按时间维度聚合的一页播放记录，包含多个分组，每个分组有时间标签和条目列表
- **ViewSettings（视图设置）**: 当前用户的界面偏好，包含：分组方式、排序方式、内容类型过滤、重复条目开关状态、视图模式（thumbnail/poster/list）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户从点击插件入口到看到播放记录列表的时间不超过 3 秒（正常网络条件下）
- **SC-002**: 切换分组/排序/过滤选项后，新结果在 2 秒内完成刷新
- **SC-003**: 分组标签和日期显示格式 100% 符合需求定义（含季节名称、周范围等）
- **SC-004**: "显示重复条目"开关功能经测试，在各种场景下行为与需求描述 100% 一致
- **SC-005**: 翻页功能正确，不出现条目重复或遗漏（可通过已知数据集验证）
- **SC-006**: 仅当前用户的播放记录可见，切换不同账号登录后显示对应用户记录（隔离性验证）
- **SC-007**: 插件在 Jellyfin 10.8+ 版本的 Web 客户端中可正常安装、访问、使用
- **SC-008**: 用户仅通过添加仓库 URL 并点击安装，可在 5 分钟内完成插件安装并进入使用状态

## Assumptions

- 目标用户为 Jellyfin Web（浏览器访问）的使用者，不包括原生移动/TV 客户端
- Jellyfin 服务器版本为 10.8 或更高（支持插件页面注册机制）
- 插件维护独立的 SQLite 数据库（位于 Jellyfin 配置目录下），记录播放开始事件和收藏变更事件，不依赖 Jellyfin Activity Log 的保留策略；数据库 schema 通过 `PRAGMA user_version` 版本控制，每次启动时自动执行增量迁移
- 播放事件通过 Jellyfin 事件系统订阅（`IEventConsumer<PlaybackStartEventArgs>`），收藏事件通过 `IEventConsumer<UserDataSaveEventArgs>` 订阅，数据实时写入 SQLite
- 媒体标题、缩略图等展示性元数据在前端通过 Items API 批量补全，不存入数据库（避免冗余，保持与 Jellyfin 元数据同步）
- "收藏时间"排序基于数据库记录的 FavoritedAt 时间戳（最近收藏的排前），取消收藏后 FavoritedAt 清为 null，排至末尾
- "按周"分组中，跨年周以该周起始日所在年份归属
- 空的时间分组（该周期内无播放记录）不显示，直接跳过
- 翻页为手动翻页（点击按钮），不支持无限滚动
- 首期版本不包括搜索/筛选特定标题的功能
- 设置状态仅保存在浏览器本地（localStorage），不同步到服务端或其他设备
- 插件通过 GitHub Releases 分发，manifest.json 托管于 GitHub Pages，用户添加该 URL 作为 Jellyfin 第三方仓库
- 插件最低兼容 Jellyfin 10.8.0（targetAbi），开发环境基于 10.11.x
