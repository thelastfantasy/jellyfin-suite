# Feature Specification: Web Player Enhancer

**Feature Branch**: `feature/005-player-enhancer`  
**Created**: 2026-05-18  
**Status**: Draft

## Clarifications

### Session 2026-05-18

- Q: 亮度调节状态在视频切换后如何处理？ → A: 每次新视频开始时重置为 100%，不持久化
- Q: 移动端 OSD 隐藏时手势是否仍生效？ → A: 是，手势始终生效，OSD 是纯显示层，与手势操作无关（同哔哩哔哩、虎牙等 App 实现）
- Q: 注入管理 UI（状态显示 + 重新注入/卸载按钮）是否应写入 spec？ → A: 是，加入 spec 作为用户故事与功能需求
- Q: 亮度调节上限是多少？ → A: 0%–200%，100% 为基准正常亮度，最高可提亮至 200%
- Q: 注入操作失败时如何反馈用户？ → A: 弹出错误提示并附加"查看服务端日志"引导，按钮恢复正常状态，注入状态不变
- Q: "含字幕"开关是否持久化？默认值是什么？ → A: 不持久化，每次播放器初始化重置；默认值为关闭
- Q: 注入管理 UI 和 API 的权限范围？ → A: 仅管理员可见且可调用；config.json 是服务器全局文件，修改影响所有用户，属管理员配置决策而非用户偏好
- Q: 非支持语言的 i18n 回退策略？ → A: 按 navigator.language 最优匹配回退（如 zh-TW → 中文，fr-FR → 英语）

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 帧步进控制 (Priority: P1)

视频分析用户（影评人、字幕校对员、视频制作者）在播放界面直接逐帧检视画面，无需借助外部播放器。播放器底部新增四个步进按钮：后退 10 帧、后退 1 帧、前进 1 帧、前进 10 帧。

**Why this priority**: 这是本功能中最独特、市场上 Jellyfin 原生及同类插件均不具备的能力，直接满足精确画面分析的核心需求。

**Independent Test**: 打开任意视频进入播放器，在 OSD 底部可见四个步进按钮，点击后视频画面精确移动对应帧数，即可独立验证。

**Acceptance Scenarios**:

1. **Given** 视频正在播放，**When** 用户点击任意步进按钮，**Then** 视频自动暂停并移动到对应帧位置
2. **Given** 视频已暂停，**When** 用户点击前进 1 帧，**Then** 画面前进一帧并保持暂停
3. **Given** 视频已暂停，**When** 用户点击后退 10 帧，**Then** 画面后退十帧并保持暂停
4. **Given** 视频处于第 1 帧，**When** 用户点击后退按钮，**Then** 画面停留在第 1 帧不越界
5. **Given** 视频处于最后一帧，**When** 用户点击前进按钮，**Then** 画面停留在最后一帧不越界

---

### User Story 2 - 截图保存 (Priority: P2)

用户在播放界面一键保存当前画面为图片文件，可选是否将屏幕上显示的字幕一并合入截图。

**Why this priority**: 截图功能在任何设备上均可使用，受众广，使用门槛低，可独立交付价值。

**Independent Test**: 播放任意视频暂停，点击截图按钮，浏览器自动下载一张 PNG 图片，图片内容与当前暂停画面一致，即可独立验证。

**Acceptance Scenarios**:

1. **Given** 视频播放或暂停中，**When** 用户点击截图按钮，**Then** 浏览器下载一张与当前画面内容一致的 PNG 图片，不包含浏览器界面或系统 UI
2. **Given** 当前画面有 ASS/SSA 格式字幕显示且字幕 Switch 开启，**When** 用户点击截图按钮，**Then** 下载的图片中包含可见的字幕文字
3. **Given** 当前画面无字幕，**When** 用户截图（无论是否选择含字幕），**Then** 截图正常完成，无报错
4. **Given** DRM 保护内容，**When** 用户点击截图，**Then** 显示"此内容受版权保护，无法截图"提示，不产生空白图片

---

### User Story 3 - 移动端双击手势（快进/快退/暂停恢复）(Priority: P3)

移动设备用户通过双击屏幕的左、中、右三个区域，分别触发快退 10 秒、暂停/恢复播放、快进 10 秒，操作体验与主流视频 App（爱奇艺、优酷等）一致。

**Why this priority**: 移动端触控操作体验的基础改善，受益人群为所有移动端用户，与 P4 滑动控制共同构成完整的移动端增强体验。

**Independent Test**: 在移动设备上打开任意视频，双击屏幕左侧退 10 秒、双击中间暂停/恢复、双击右侧进 10 秒；仅在移动设备上生效，即可独立验证。

**Acceptance Scenarios**:

1. **Given** 移动设备用户正在观看视频，**When** 双击屏幕左侧 1/3 区域，**Then** 播放进度后退 10 秒并显示短暂提示动画
2. **Given** 移动设备用户正在观看视频，**When** 双击屏幕右侧 1/3 区域，**Then** 播放进度前进 10 秒并显示短暂提示动画
3. **Given** 移动设备用户正在播放视频，**When** 双击屏幕中间 1/3 区域，**Then** 视频暂停，无提示动画
4. **Given** 移动设备用户视频已暂停，**When** 双击屏幕中间 1/3 区域，**Then** 视频恢复播放，无提示动画
5. **Given** 桌面端用户，**When** 在播放器上双击，**Then** 行为与原版 Jellyfin 保持一致（不触发任何手势功能）
6. **Given** 快进超出视频末尾，**When** 双击右侧，**Then** 跳转至视频最后并停止，不报错

---

### User Story 4 - 移动端滑动调节亮度与音量 (Priority: P4)

移动设备用户通过在屏幕左半侧上下滑动调节画面亮度，在右半侧上下滑动调节播放音量，滑动过程中屏幕上显示当前值的进度指示。

**Why this priority**: 作为 P3 双击手势的配套功能，共同完善移动端触控交互；优先级低于 P3 是因为 Jellyfin 已有音量滑块，此功能属于便捷性增强。

**Independent Test**: 在移动设备上播放视频，在屏幕左侧向上滑动，画面变亮并显示亮度百分比；在右侧向上滑动，音量增大并显示音量百分比，即可独立验证。

**Acceptance Scenarios**:

1. **Given** 移动设备用户正在观看视频，**When** 在屏幕左侧向上滑动，**Then** 画面逐渐变亮，屏幕上显示当前亮度百分比
2. **Given** 移动设备用户正在观看视频，**When** 在屏幕左侧向下滑动，**Then** 画面逐渐变暗，屏幕上显示当前亮度百分比
3. **Given** 移动设备用户正在观看视频，**When** 在屏幕右侧向上滑动，**Then** 音量增大，屏幕上显示当前音量百分比
4. **Given** 移动设备用户正在观看视频，**When** 在屏幕右侧向下滑动，**Then** 音量减小，屏幕上显示当前音量百分比
5. **Given** 亮度或音量已达最大/最小值，**When** 继续滑动，**Then** 数值保持边界值不越界，指示器停止在顶端/底端
6. **Given** 桌面端用户，**When** 在播放器上拖拽，**Then** 滑动调节功能不生效，不干扰桌面端操作

---

### User Story 5 - 播放器增强注入管理 (Priority: P5)

Jellyfin 管理员在"最近播放"设置面板中查看播放器增强的当前注入状态，并可手动执行重新注入或卸载注入，无需重启服务器。注入管理区域仅对管理员可见，普通用户不受影响。

**Why this priority**: 自动注入覆盖正常运行场景；手动管理 UI 面向 Jellyfin 升级后 config.json 被重置但服务尚未重启、或希望临时禁用增强功能的管理员，是一个低频但必要的控制入口。config.json 是服务器全局文件，修改影响所有用户，属管理员权限范畴。

**Independent Test**: 以管理员身份打开"最近播放"设置面板，可见注入状态标签及两个操作按钮；以普通用户身份打开面板，注入管理区域不可见；点击"重新注入"后状态变为已注入，点击"卸载注入"后状态变为已禁用，即可独立验证。

**Acceptance Scenarios**:

1. **Given** 管理员打开"最近播放"设置面板，**When** 页面加载完成，**Then** 显示注入管理区域及当前注入状态（已注入 / 已禁用）；普通用户打开同一面板时注入管理区域不可见
2. **Given** 注入状态为已禁用（如 Jellyfin 升级后），**When** 管理员点击"重新注入"，**Then** 注入操作执行，状态更新为已注入，并提示刷新页面生效
3. **Given** 注入状态为已注入，**When** 管理员点击"卸载注入"，**Then** 注入被移除，状态更新为已禁用，提示刷新页面生效；此后服务重启不自动恢复注入
4. **Given** 管理员点击任意操作按钮，**When** 操作执行中，**Then** 按钮显示加载状态，防止重复点击

### Edge Cases

- 视频文件元数据缺失帧率信息时，帧步进以常见默认值工作，并在 OSD 区域提示用户
- 用户在视频缓冲/加载中点击步进按钮，等待缓冲完成后执行步进
- 截图时视频处于旋转/翻转状态（部分移动端场景），截图方向应与屏幕显示一致
- 字幕同时存在多个轨道时，截图仅合入当前激活的字幕轨道
- 滑动手势与 Jellyfin 原生 OSD 显示/隐藏手势（单击）发生冲突时，优先触发滑动调节
- 双击手势恰好落在 OSD 按钮区域时，应优先触发按钮点击而非双击快进/快退
- 截图时若视频亮度已被滑动手势调节（非 100%），Canvas `drawImage` 读取的是 `<video>` 元素未经 CSS filter 处理的原始像素，截图亮度与屏幕显示不一致；v1 为已知限制，不作为缺陷处理

## Requirements *(mandatory)*

### Functional Requirements

**帧步进**

- **FR-001**: 播放器 OSD 底部 MUST 新增四个步进控制按钮，功能分别为后退 10 帧、后退 1 帧、前进 1 帧、前进 10 帧
- **FR-002**: 步进操作 MUST 在视频播放和暂停两种状态下均可触发；播放状态下触发时自动暂停后执行步进
- **FR-003**: 步进精度 MUST 基于当前视频的实际帧率，非固定时间偏移
- **FR-004**: 步进操作在到达视频首帧或末帧时 MUST 停止，不得产生负时间戳或超出时长的定位

**截图**

- **FR-005**: 播放器 OSD 底部 MUST 新增截图按钮，及相邻的"含字幕"Switch 开关（默认关闭；每次播放器初始化重置，不跨会话持久化）
- **FR-006**: 截图结果 MUST 仅包含视频画面内容，不含浏览器窗口边框、标签栏或操作系统 UI；播放器 CSS 产生的黑边（letterbox）不进入截图
- **FR-007**: "含字幕"Switch 开启时，截图 MUST 叠加当前激活的 ASS/SSA canvas 字幕；SRT/VTT 原生轨道字幕因浏览器限制无法捕获，Switch 对其静默无效（不报错）
- **FR-008**: 截图文件 MUST 以原始视频编码分辨率输出为 PNG 格式，自动下载到用户本地
- **FR-009**: 当内容受版权保护无法截图时，MUST 显示明确的用户提示，不得静默产生空白图片

**移动端双击**

- **FR-010**: 在触控设备上，双击播放区域左侧 1/3 MUST 使播放进度后退 10 秒
- **FR-011**: 在触控设备上，双击播放区域右侧 1/3 MUST 使播放进度前进 10 秒
- **FR-012**: 在触控设备上，双击播放区域中间 1/3 MUST 切换暂停/恢复播放状态，不显示动画
- **FR-013**: 双击左/右区域触发时 MUST 显示方向与秒数提示动画（ripple 效果）；双击中间区域不显示动画
- **FR-023**: 所有双击手势的触控设备限定同 FR-018（`navigator.maxTouchPoints > 0`），不影响桌面端鼠标双击行为

**移动端滑动**

- **FR-014**: 在触控设备上，在播放区域左半侧上下滑动 MUST 调节画面亮度
- **FR-015**: 在触控设备上，在播放区域右半侧上下滑动 MUST 调节播放音量
- **FR-016**: 滑动过程中 MUST 在屏幕中央显示当前亮度或音量的百分比指示器
- **FR-017**: 亮度调节 MUST 在最小值（0%）和最大值（200%）之间限制（100% 为正常亮度基准，可调暗至全黑也可提亮至 200%）；音量调节 MUST 在最小值（0%）和最大值（100%）之间限制；两者均不越界
- **FR-018**: 滑动调节和双击手势 MUST 仅在触控设备上激活，不影响桌面端行为；手势始终生效，不依赖 OSD 是否可见（OSD 为纯显示层）
- **FR-019**: 亮度调节效果 MUST 作用于视频画面显示，不影响系统屏幕背光亮度；每次新视频开始播放时亮度自动重置为 100%

**通用**

**注入管理**

- **FR-024**: "最近播放"设置面板中的注入管理区域 MUST 仅对 Jellyfin 管理员显示，显示播放器增强当前注入状态（已注入 / 已禁用）；非管理员用户不可见此区域
- **FR-025**: 注入管理区域 MUST 提供"重新注入"操作（需管理员权限），执行后将增强脚本重新写入服务器全局 config.json 并启用自动注入配置；操作成功后提示管理员刷新页面生效；操作失败时 MUST 显示错误提示并附加"查看服务端日志"引导，按钮恢复可点击状态，注入状态不变
- **FR-026**: 注入管理区域 MUST 提供"卸载注入"操作（需管理员权限），执行后将增强脚本从服务器全局 config.json 中移除；操作期间按钮显示加载状态防止重复提交；操作失败时 MUST 显示错误提示并附加"查看服务端日志"引导，按钮恢复可点击状态，状态不变
- **FR-027**: "卸载注入"操作 MUST 同时持久化管理员配置决策（禁用自动注入）；后续服务重启时，EntryPoint MUST 检查此配置并跳过自动注入，直到管理员主动执行"重新注入"

**通用**

- **FR-020**: Jellyfin 服务重启后，若管理员未执行过"卸载注入"（即自动注入配置为启用），增强脚本 MUST 自动恢复注入，无需人工介入；若管理员已执行"卸载注入"，则服务重启后不得自动恢复
- **FR-021**: 新增功能 MUST 不破坏 Jellyfin 原有播放器的任何现有功能
- **FR-022**: 所有用户可见文字（按钮 tooltip、OSD 标签、提示信息）MUST 支持中文、日语、英语三语，根据 Jellyfin 界面语言设置自动切换；不支持的语言按 `navigator.language` 最优匹配回退（如 `zh-TW` → 中文，`fr-FR` → 英语）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户可在播放界面不离开 Jellyfin 的情况下，在 3 秒内完成单帧步进操作
- **SC-002**: 截图操作从点击到文件下载完成，在正常网络条件下不超过 2 秒
- **SC-003**: 移动端三区域双击手势（快退/暂停恢复/快进）的识别成功率不低于 95%（不与单击/滑动混淆）
- **SC-004**: 移动端滑动调节亮度/音量的响应延迟感知不超过 100ms（即时跟手）
- **SC-005**: Jellyfin 升级后，下次服务重启时，在管理员未执行"卸载注入"的情况下，增强功能自动恢复，无需人工介入；管理员的"卸载注入"配置决策跨重启持久化
- **SC-006**: 新增功能不导致播放器原有功能（字幕、音轨切换、进度拖拽等）的任何回归

## Assumptions

- 目标 Jellyfin 版本为 10.10.x，功能基于该版本的播放器界面结构设计
- 亮度调节仅调整视频画面的视觉呈现效果，不控制设备屏幕的物理背光（Web 平台限制）
- 帧步进功能面向桌面和移动端所有用户；双击与滑动手势仅面向触控设备用户
- 截图功能在 DRM 加密内容上因浏览器安全限制可能不可用，此为预期行为而非缺陷
- 插件已安装在 Jellyfin 服务器上，用户通过浏览器访问 Jellyfin Web 界面使用上述功能
- 视频帧率信息可从 Jellyfin 媒体库元数据中获取；若缺失则使用 24fps 作为默认值
